Fase 01

Business Case & Data Collection

01
Business Understanding
Identifica el problema de negocio que quieres resolver
▾

Antes de tocar ningún dato, define con precisión qué pregunta intentas responder. Un EDA sin objetivo es solo una exploración sin dirección. Escribe una problem statement en una sola frase.

💡 Tip: «¿Qué variables explican la tasa de abandono de clientes en los últimos 6 meses?» es un buen punto de partida. Evita objetivos vagos como «entender los datos».
  • Problem statement definida en una frase
  • Stakeholders identificados
  • Métricas de éxito establecidas
02
Hipótesis y plan de acción
Formula afirmaciones comprobables sobre los datos
▾

Formula hipótesis concretas y define qué harás si se confirman o se refutan. Sin un plan de acción, los hallazgos no generan valor.

💡 Ejemplo: H1: «Los clientes con antigüedad < 6 meses tienen el doble de tasa de abandono». Si es verdad → campaña de retención temprana. Si es falso → buscar otras variables.
  • Al menos 3 hipótesis formuladas
  • Plan de acción por cada hipótesis
03
Requerimientos de los datos
Lista los datos necesarios a alto nivel
▾

Identifica qué datos necesitas: granularidad, rango temporal, fuentes, y variables mínimas imprescindibles para validar tus hipótesis.

  • Fuentes de datos listadas
  • Granularidad y ventana temporal definidas
  • Variables mínimas necesarias identificadas
04
Disponibilidad
Primer punto de inflexión del proyecto
▾

Verifica si los datos existen y si tienes acceso legal y técnico a ellos. Si no es posible, replantea el proyecto antes de invertir más tiempo.

  • Datos accesibles confirmado
  • Permisos y licencias revisados
05
Adquisición de datos Data Acquisition
Carga datos desde distintas fuentes
▾

Carga los datos desde su origen: CSV, base de datos, API, web scraping o dataset público.

Python · Pandas
import pandas as pd # Desde CSV df = pd.read_csv('datos.csv', encoding='utf-8') # Desde Excel df = pd.read_excel('datos.xlsx', sheet_name='Sheet1') # Desde base de datos SQL import sqlalchemy engine = sqlalchemy.create_engine('postgresql://user:pwd@host/db') df = pd.read_sql("SELECT * FROM clientes", engine) # Desde API (ejemplo JSON) import requests response = requests.get('https://api.ejemplo.com/datos') df = pd.DataFrame(response.json())
  • Datos cargados correctamente en DataFrame
  • Backup del raw data guardado
06
Calidad de los datos Data Quality
Segundo punto de inflexión del proyecto
▾

Evalúa si los datos son suficientes, válidos y representativos. Demasiados missings o datos manipulados pueden invalidar el análisis.

Python · Pandas
# Resumen rápido de calidad def calidad_datos(df): resumen = pd.DataFrame({ 'dtype': df.dtypes, 'nulos': df.isnull().sum(), 'pct_nulos': (df.isnull().sum() / len(df) * 100).round(2), 'unicos': df.nunique() }) return resumen print(calidad_datos(df))
07
Revisión de hipótesis
Ajusta los objetivos al dato real disponible
▾

Una vez vistos los datos reales, ajusta o reformula las hipótesis. Es normal tener que pivotarlas: los datos mandan.

  • Hipótesis revisadas contra datos disponibles
  • Variables descartadas o añadidas

09
Tabla de variables Data Dictionary
Inventaria y documenta cada columna
▾

Documenta qué es cada variable, su tipo, si es continua/categórica, y qué crees que va a aportar. Una hoja de cálculo funciona perfectamente.

Python · Pandas
# Genera un data dictionary automático data_dict = pd.DataFrame({ 'columna': df.columns, 'dtype': df.dtypes.values, 'nulos_pct': (df.isnull().mean() * 100).round(1).values, 'unicos': df.nunique().values, 'ejemplo': [df[c].dropna().iloc[0] if len(df[c].dropna()) > 0 else None for c in df.columns] }) data_dict.to_csv('data_dictionary.csv', index=False) data_dict
Fase 03

Data Cleaning

10
Unión de tablas Merge / Join
Consolida fuentes de datos en una sola tabla
▾

Si tus datos vienen de varias fuentes, únalos usando la clave común. Elige el tipo de join que tenga sentido para tu análisis.

Python · Pandas
# Inner join — solo registros comunes df_merged = pd.merge(df1, df2, on='id_cliente', how='inner') # Left join — todos los de df1 aunque no estén en df2 df_merged = pd.merge(df1, df2, on='id_cliente', how='left') # Apilar filas del mismo esquema df_total = pd.concat([df_2023, df_2024], ignore_index=True) print(f"Registros resultado: {len(df_merged):,}")
11
Eliminar columnas Feature Selection
Quita variables que no aporten al análisis
▾

Elimina identificadores únicos, columnas constantes, columnas con demasiados missings y variables irrelevantes para las hipótesis.

Python · Pandas
# Eliminar columnas específicas df.drop(columns=['id', 'timestamp_interno'], inplace=True) # Eliminar columnas con > 60% de missings umbral = 0.6 cols_a_eliminar = df.columns[df.isnull().mean() > umbral] df.drop(columns=cols_a_eliminar, inplace=True) print(f"Columnas eliminadas: {list(cols_a_eliminar)}") # Eliminar columnas constantes (una sola categoría) cols_constantes = [c for c in df.columns if df[c].nunique() == 1] df.drop(columns=cols_constantes, inplace=True)
12
Formato de los datos Data Types
Fuerza los tipos correctos en cada columna
▾

Un número cargado como string o una fecha como object generará errores silenciosos en todo el análisis posterior. Corrígelo antes.

Python · Pandas
# Convertir a numérico (coerce → NaN si no puede) df['precio'] = pd.to_numeric(df['precio'], errors='coerce') # Convertir fechas df['fecha_compra'] = pd.to_datetime(df['fecha_compra'], format='%Y-%m-%d') # Convertir a categórico (ahorra memoria) df['segmento'] = df['segmento'].astype('category') # Verificar resultado df.dtypes
13
Valores nulos Missing Values
Analiza e imputa los nulos con criterio
▾

Antes de imputar, entiende por qué hay nulos. En Data Analysis, un nulo puede ser un insight en sí mismo (ej: campo de devolución vacío = no hubo devolución).

Python · Pandas
# Visualizar missings import missingno as msno msno.matrix(df) # Imputar con la mediana (variables numéricas) df['edad'].fillna(df['edad'].median(), inplace=True) # Imputar con la moda (variables categóricas) df['ciudad'].fillna(df['ciudad'].mode()[0], inplace=True) # Eliminar filas con nulos en columnas clave df.dropna(subset=['target', 'fecha_compra'], inplace=True)
14
Valores atípicos Outliers
Detecta y decide qué hacer con los extremos
▾

En Data Analysis los outliers son hallazgos, no ruido. Detéctalos, investiga su causa y decide si son errores o valores legítimos.

Python · Pandas + Seaborn
import seaborn as sns import matplotlib.pyplot as plt # Método IQR Q1 = df['precio'].quantile(0.25) Q3 = df['precio'].quantile(0.75) IQR = Q3 - Q1 lower = Q1 - 1.5 * IQR upper = Q3 + 1.5 * IQR outliers = df[(df['precio'] < lower) | (df['precio'] > upper)] print(f"Outliers detectados: {len(outliers)}") # Boxplot sns.boxplot(x=df['precio']) plt.show()
15
Duplicados Duplicates
Elimina registros repetidos
▾
Python · Pandas
# Verificar duplicados n_dups = df.duplicated().sum() print(f"Duplicados encontrados: {n_dups}") # Eliminar duplicados exactos df.drop_duplicates(inplace=True) # Duplicados por columna clave (ej: un cliente no puede aparecer 2x) df.drop_duplicates(subset=['id_cliente'], keep='last', inplace=True)
16
Escalado Scaling
Normaliza variables con escalas distintas
▾

Si vas a comparar o graficar variables con escalas muy distintas (ej: ingresos en €10.000 vs. edad en 0-100), normaliza con MinMaxScaler.

Python · Scikit-learn
from sklearn.preprocessing import MinMaxScaler scaler = MinMaxScaler() cols_numericas = ['ingresos', 'edad', 'num_compras'] df[cols_numericas] = scaler.fit_transform(df[cols_numericas]) # Ahora todas las variables están entre 0 y 1
17
Transformaciones Feature Engineering
Crea nuevas variables a partir de las existentes
▾

Extrae año, mes y día de una fecha. Crea ratios. Aplica binning para convertir variables continuas en categorías.

Python · Pandas
# Extraer componentes de fecha df['anyo'] = df['fecha_compra'].dt.year df['mes'] = df['fecha_compra'].dt.month df['dia_semana'] = df['fecha_compra'].dt.day_name() # Ratio numérico df['ticket_medio'] = df['ingresos'] / df['num_compras'] # Binning — convertir edad continua en tramos df['tramo_edad'] = pd.cut( df['edad'], bins=[0, 25, 35, 50, 65, 100], labels=['18-25', '26-35', '36-50', '51-65', '65+'] )
Fase 04

Analysis

18
Análisis univariante Univariate Analysis
Entiende cada variable por separado
▾

Analiza la distribución, centralidad y dispersión de cada variable. Usa histogramas y boxplots para las numéricas, y gráficos de barras para las categóricas.

Python · Pandas + Matplotlib
# Estadísticos de centralidad y dispersión stats = df['precio'].agg(['mean', 'median', 'std', 'min', 'max']) print(stats) # Histograma + KDE import matplotlib.pyplot as plt fig, axes = plt.subplots(1, 2, figsize=(12, 4)) df['precio'].hist(bins=30, ax=axes[0]) sns.boxplot(x=df['precio'], ax=axes[1]) plt.show() # Variable categórica df['segmento'].value_counts().plot(kind='bar') plt.show()
19
Análisis bivariante Bivariate Analysis
Descubre relaciones entre pares de variables
▾

Cruza dos variables para descubrir patrones. Agrupa por categóricas y calcula estadísticos. Usa scatter plots para relaciones numéricas.

Python · Pandas + Seaborn
# Agrupado: media de precio por segmento df.groupby('segmento')['precio'].agg(['mean', 'count', 'std']) # Scatter plot dos variables numéricas sns.scatterplot(data=df, x='edad', y='precio', hue='segmento', alpha=0.6) # Boxplot agrupado (numérico vs categórico) sns.boxplot(data=df, x='segmento', y='precio') # Cross table (dos categóricas) pd.crosstab(df['segmento'], df['tramo_edad'], normalize='index')
20
Análisis multivariante Multivariate Analysis
Correlación y patrones entre todas las variables
▾

El heatmap de correlación y el pairplot son tus herramientas clave para ver de un vistazo las relaciones entre todas las variables numéricas.

Python · Seaborn
# Heatmap de correlación plt.figure(figsize=(12, 8)) corr_matrix = df.select_dtypes(include='number').corr() sns.heatmap( corr_matrix, annot=True, fmt='.2f', cmap='coolwarm', center=0, square=True ) plt.show() # Pairplot — relación de todas con todas sns.pairplot(df, hue='segmento', diag_kind='kde', plot_kws={'alpha': 0.4}) plt.show()

▾

Vuelve a tus hipótesis iniciales y respóndelas una a una con evidencia de los datos. Sé honesto: si el análisis no responde la pregunta de negocio, toca iterar.

  • H1 confirmada / refutada con evidencia
  • H2 confirmada / refutada con evidencia
  • H3 confirmada / refutada con evidencia
  • Insights inesperados documentados
22
Plan de acción
¿Qué se hace con los resultados?
▾

Traduce cada hallazgo en una acción concreta. Un análisis sin plan de acción es un gasto de tiempo, no una inversión.

  • Acciones priorizadas por impacto
  • Responsables asignados
  • Timeline definido
23
Productivización Productionization
¿El análisis necesita vivir en un dashboard?
▾

Si el análisis debe ser consultable de forma recurrente por otros equipos, plantéate productivizarlo en Tableau, Power BI, Looker o un dashboard en Python (Streamlit, Dash).

  • Necesidad de dashboard evaluada
  • Herramienta de BI seleccionada
24
Comunicación de resultados Storytelling
Presenta con hilo conductor claro
▾

Una buena presentación sigue la estructura: problema → datos → proceso → hallazgos → acciones. Tu audiencia no entiende de datos, adapta el lenguaje.

💡 Regla de oro: si no puedes explicar tu hallazgo en dos frases al CEO, no lo entiendes tú tampoco. Simplifica.
  • Presentación con estructura narrativa
  • Gráficas limpias y sin ruido visual
  • Conclusiones en lenguaje de negocio
